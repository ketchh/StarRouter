Install in one minute

Install from npm

pi install npm:pi-star-router /reload /router settings /router on

StarRouter is a normal Pi package declared through package.json and loaded as an extension.

Try it once

pi -e npm:pi-star-router

Use a one-off run when you want to test routing without installing the package globally.

Why StarRouter exists

Cheap tasks should stay cheap. Hard tasks should get the model they deserve. Every switch should be explainable.

Cost control

Mechanical transforms, JSON extraction, simple rewrites, and low-risk coding can route to efficient models instead of frontier defaults.

Quality guardrails

Debugging, architecture, long-context, vision, and high-stakes research get stricter thinking and benchmark constraints.

User control

Auto-accept is off by default. The confirmation panel shows candidates and lets the user keep the current route.

Routing pipeline

Core features

Deterministic heuristic prompt profiler

Local English and Italian dictionaries detect mechanical transforms, simple output, coding, debugging, architecture, security, research, long context, and vision. No hidden classifier model is called before your turn.

Provider-scoped routing

StarRouter compares only models from the configured provider. This makes every recommendation easier to reason about and avoids accidental cross-provider surprises.

Strict AA model matching

Vendor, family, subfamily, generation, moving aliases, host affinity, and reasoning variant all matter before a Pi model can borrow an Artificial Analysis row.

Supported thinking levels only

Candidates are generated only for thinking levels exposed by Pi model metadata, preventing impossible model@thinking recommendations.

Benchmark-safe filters

Built-in presets keep large provider catalogs manageable by excluding auto/free/noisy endpoints while preserving benchmarkable families.

Explainable confirmation UI

The route panel shows the current model, recommended route, top candidates, fit/cost/latency/context scores, confidence, and short reason bits.

Trust model

Conservative by default

  • Disabled until /router on.
  • Confirmation required unless auto-accept is enabled.
  • Abstains on low confidence.
  • Does not use hidden model-assisted classification.

Correctness before cleverness

  • Strict model-family matching.
  • Moving aliases require explicit override.
  • Vision and context are hard constraints.
  • Current route is kept when alternatives are only marginally better.
