Conclusion

No known automated release blocker remains. Routing is provider-scoped, local, deterministic, explicit by default, and capable of abstaining. Model-only aggregator evidence is separated from host-verified evidence, so another host’s price or performance cannot leak into an OpenRouter decision.

Attestation boundary

Static render matrices cover 120×40 through 40×6. Real terminal emulators, CJK IME placement, RPC hosts, and screen readers still require human/device QA.

The decisions that matter.

Evidence is typed, not implied.

A host mismatch cannot become verified through a model API ID. Aggregators retain model-quality evidence with an explicit confidence penalty.

05 / FINDING LEDGER

Fixed blockers and honest residuals.

Resolved before release

Fixed
Override identity bypass.
Alias pins now retain creator, family, version, reasoning, and evidence-scope gates; hostSlug is not a model target.
Fixed
False host certification.
hostApiId no longer certifies host scope, closing Azure/Vertex metric leakage.
Fixed
OpenRouter zero-route failure.
Aggregator model-only evidence restores quality matching without borrowing unrelated host performance.
Fixed
Non-standard Pareto filtering.
Standard weak dominance with numerical epsilon preserves real trade-offs.
Fixed
Refresh/cache inconsistency.
Forced refresh bypasses fresh disk cache, reports stale provenance, and obsolete generations cannot overwrite newer cache.
Fixed
Transactional and lifecycle ambiguity.
Save/Cancel drafts, persisted clear markers, RPC-safe widgets, print guidance, and disabled-widget cleanup now agree.

Accepted / manual residuals

Manual
Physical terminal QA.
Confirm overlay borders, keyboard protocols, CJK IME placement, and optical density in real emulators.
External
Catalog drift.
AA and OpenRouter schemas, models, hosts, and prices can change independently; scheduled live checks are designed to fail visibly.
Durability
Directory fsync.
Config writes use temp-file rename but do not fsync the parent directory after commit.
Scope
Offline guard boundary.
The test preload is not OS-level network isolation and does not contain malicious child processes or native modules.
